# LangGraph Agent 分支

## 分支目标

在 FastAPI AI 服务基础上，实现能够调用真实业务接口的客服 Agent。

## 当前状态

未开始。

## 已确定方案

- Agent 不只回答文档问题，还要能调用 Java 后端业务接口。
- Java 后端提供订单查询、商品库存查询等工具接口。
- LangGraph 负责编排意图判断、工具调用和最终回答。

## 计划工具

- 订单查询工具。
- 商品库存查询工具。
- 售后政策查询工具。
- 后期可扩展售后工单创建工具。

## 初始流程

```text
用户问题
 -> 判断是否需要查知识库
 -> 判断是否需要调用业务工具
 -> 调用 Java 后端接口
 -> 汇总业务结果和知识库内容
 -> 生成客服回答
```

## 待解决问题

1. Agent 状态如何设计。
2. 工具调用失败如何处理。
3. Java 后端接口鉴权如何做。
4. 多轮对话记忆是否第一版就加入。

## 当前任务

等待 FastAPI RAG 服务完成后再进入。

## 与其他分支的关联

- FastAPI AI 服务分支提供基础服务。
- Java 后端分支提供业务查询接口。
- API 设计分支定义工具接口契约。
- 简历分支会包装本模块作为 AI 应用亮点。
