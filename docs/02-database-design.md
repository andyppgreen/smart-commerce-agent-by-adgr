# 数据库设计分支

## 分支目标

设计智能电商运营平台的 MySQL 数据模型，为 Java 后端开发提供稳定基础。

## 当前状态

未开始。

## 已确定方案

- 使用 MySQL。
- 前期围绕用户、角色、商品、分类、订单、秒杀活动建模。
- 表设计优先服务 MVP，不做过度复杂的电商模型。

## 初始候选表

- `sys_user`：用户表。
- `sys_role`：角色表。
- `sys_user_role`：用户角色关联表。
- `product_category`：商品分类表。
- `product`：商品表。
- `order_info`：订单表。
- `order_item`：订单明细表。
- `seckill_activity`：秒杀活动表。
- `seckill_order`：秒杀订单表。

## 待解决问题

1. 用户是否区分后台管理员和普通买家。
2. 商品库存是否单独拆表。
3. 订单状态流转如何设计。
4. 秒杀订单和普通订单如何关联。

## 当前任务

设计第一版 MVP 数据库表结构。

## 与其他分支的关联

- Java 后端分支需要基于本分支生成实体类、Mapper、Service。
- Redis 秒杀分支需要引用商品、库存、秒杀活动和订单表。
- LangGraph Agent 分支需要读取订单、商品等业务数据。
