# ADR-001：前期采用单体 Java 后端 + 独立 Python AI 服务

## 背景

项目目标是训练 Java 后端、Redis、消息队列、FastAPI 和 Agent 工程化能力。当前阶段更需要完成一个可运行、可理解、可展示的系统，而不是一开始引入过多复杂架构。

## 决策

前期采用：

- Java Spring Boot 单体后端承载电商业务。
- Python FastAPI 独立服务承载 AI 客服与 Agent 能力。
- Java 后端和 Python AI 服务通过 HTTP 接口通信。

暂不引入 Spring Cloud 微服务体系。

## 原因

1. 降低早期复杂度。
2. 优先训练基础业务建模、接口开发、数据库设计和缓存能力。
3. AI 服务天然适合用 Python 独立开发。
4. 后期如果需要，可以把订单、秒杀、AI 服务逐步拆分。

## 后续演进

当基础功能稳定后，可以考虑：

- 拆分订单服务。
- 拆分秒杀服务。
- 引入网关。
- 引入服务注册与配置中心。
- 增加链路追踪和监控。
